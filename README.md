### 🧠 Arquitetura e decisões técnicas

- APIs orientadas a consumo com controle de billing por requisição  
- Rastreabilidade ponta a ponta utilizando `request_id`  
- Separação de responsabilidades (API, regras de negócio e interface)  
- Estratégias de retry para tolerância a falhas externas  
- Logging estruturado para auditoria e debugging  
- Integração entre múltiplas camadas (Node.js + PHP + WordPress)  

---

### ⚠️ Desafios técnicos enfrentados

- Integração com sistemas de terceiros sujeitos a mudanças (web scraping com autenticação)  
- Tratamento de instabilidade e indisponibilidade externa  
- Garantia de consistência em controle de consumo concorrente  
- Manipulação e proteção de dados sensíveis  
- Automação de processos dependentes de fontes externas
